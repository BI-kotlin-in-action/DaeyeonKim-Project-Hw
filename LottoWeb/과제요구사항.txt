이전에 했던 로또 프로젝트를 웹 어플리케이션으로 옮겨 봅시다

필수 요구사항
1. 유저가 여러명이 존재 할수 있습니다
2. 한 유저는 돈을 충전할수 있고 서버는 충전 금액을 기억해야 합니다
    1. 유저별로 금액을 따로 관리해야 합니다
3. 수동로또/자동로또는 사용자가 구매하고 싶은 만큼만 구매해야 합니다
    1. Ex) 로또 100장을 구매할 돈이 있어도 사용자가 수동 1개, 자동 1개만 구매하면 나머지 돈은 킵
4. 충전한 돈은 출금도 할 수 있습니다
5. 승리 로또는 10분에 한번씩 바뀝니다
    1. 현재의 로또번호는 외부에서 조회하지 못하도록 해주세요
    2. 과거의 로또는 조회가능합니다
    3. Ex) 현재 회차가 15회차일때 사용자가 13회차 로또 번호 조회시 조회 가능, 15회차 번호 요구시 조회 불가능
6. 모든 API는 Restful하게 작성되어야합니다
7. Read me 파일에 API명세를 작성해야합니다
8. 동시성 이슈에 신경을 써서 작성해주세요


아래는 선택 요구 사항입니다
이전과 마찬가지로 이중에 한개 이상은 적용해보셔야합니다.
또한 어떤 선택 요구사항을 적용했을지 알려주셔야합니다
1. 모든 정보를 DB에 넣고 관리해보세요
    1. In memory DB(h2 , HSQLDB, derby)도 좋고 상용 DBMS(mysql, oracle)도 좋습니다
2. 사용자 회원가입과 로그인 및 세션관리 등을 해보세요
3. 로그 시스템을 만들어서 자신의 로그뿐만 아니라 타인의 로또 구매 로그 및 당첨 로그도 볼수 있도록 해보세요
4. 스웨거 혹은 RestDoc과 같은 API 문서화 툴을 써보세요
5. 당첨 시스템을 현실세계와 똑같이 로또를 모았다가 한번에 맞춰보는 시스템으로 구현해보세요
6. 간단한 화면을 만들어 보세요
7. aws나 oracle cloud, azure등에 서버를 올려보세요
8. 관리자를 정의해보고 관리자는 현재의 로또 번호를 확인할수 있도록 해보세요
9. 관리자를 정의해보고 관리자가 유저를 탈퇴/ 잔고 회수 / 잔고 추가 등의 동작을 할 수 있도록 해보세요
10. MSA환경을 고려해서 개발해보세요
